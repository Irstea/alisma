ALISMA - nouveautés
===================
Version 1.2 du 5 février 2018
------------------------------
Correction de bogues :
- la date de calcul de l'IBMR (calcul SEEE) n'était pas renseignée
- modification d'un libellé dans les données mésologiques
- le calcul des coordonnées Lambert à partir des données WGS84 était erroné

Évolutions :
- le champ "longueur" est maintenant obligatoire pour valider le dossier
- rajout d'un code UUID pour faciliter les échanges inter-bases
- les coordonnées XY sont maintenant précisées en "amont"
- la saisie des coordonnées du point aval est maintenant possible
- le calcul des coordonnées WGS84 à partir des coordonnées Lambert est maintenant possible
- la largeur peut être saisie avec une décimale
- il est possible de rajouter un nouveau taxon dans la saisie du dossier en validant par la touche Entrée
- ajout d'une catégorie "autre" dans "autre type" de faciès
- modification de l'ordre de saisie des données mésologiques
- lors de la saisie du nom du taxon, le code est également affiché
- pour la turbidité, la catégorie "nulle ou faible" a été séparée en "nulle" et "faible". Les anciennes valeurs sont affectées à "faible"
- il est maintenant possible d'indiquer le code et le nom du producteur, du préleveur, du déterminateur. Les valeurs par défaut peuvent être indiquées dans le fichier param.ini
- un programme complémentaire permet de générer des fichiers csv reprenant certaines informations présentes dans le fichier XML d'export
- la recherche d'une station s'effectue maintenant en plein texte
- ajout d'une fonction d'import des cours d'eau et des taxons, à partir d'une extraction du Sandre
- l'export des dossiers au format XML peut être réalisé quel que soit l'état d'avancement du dossier
- l'affichage des indicateurs a été remodelé pour qu'il soit plus lisible
- ajout d'une fonction d'import de la liste des taxons et des codes de calcul, à partir des fichiers disponibles dans le site du SEEE
- ajout du support de la base de données HSQLDB, pour permettre un fonctionnement en mode autonome sans devoir installer MySQL
- compléments dans la documentation

Version 1.1.2 du 22 mars 2017
------------------------------
Correction de bogues :
- lors du calcul de l'indicateur, les taxons contributifs n'étaient pas correctement pris en compte
- l'interrogation directe du seee ne fonctionnait pas si le poste de travail était derrière un serveur proxy (fichier param.ini à compléter)

Version 1.1.1b du 15 mars 2017
------------------------------
Correction de bogues
- dans certains cas de figure, l'affichage des listes ne fonctionnait pas (liste réduite à une taille minimum)

Version 1.1.1 du 8 mars 2017
-------------------------------
Correction de bogues
- le script de mise à jour de la base de la version précédente (database/update-1.0-1.1.sql) contenait une instruction dupliquée, qui bloquait son exécution
- le calcul de l'IBMR était erroné dans un cas de figure : si le calcul était effectué plusieurs fois de suite, certaines valeurs n'étaient pas réinitialisées, et le résultat était incohérent
- si aucun taxon ne servait au calcul IBMR, l'indice prenait la valeur 0. Aujourd'hui, l'indice prend la valeur -1 (calcul impossible)
- le menu associé a la fenêtre de saisie d'un relevé était incomplet

Améliorations
- le calcul de l'IBMR peut être réalisé en interrogeant le service web mis à disposition par le SEEE, soit en mode "lot", soit directement en saisie d'un relevé. Dans ce dernier cas, le relevé doit être enregistré avant de pouvoir déclencher le calcul

Version 1.1 du 9 février 2017
-------------------------------
ATTENTION : une mise à jour de la base de données est nécessaire : script database/update-1.0-1.1.sql

Correction de bogues :
- dans certains cas, la recherche d'opérations échouait à cause d'une erreur SQL
- le calcul de l'IBMR, pour un ensemble de dossiers, était possiblement erroné

Améliorations
- il est maintenant possible d'exporter les données des relevés pour réaliser le calcul de l'IBMR auprès de SEEE http://www.seee.eaufrance.fr (calcul officiel), puis d'importer les résultats. Le statut des dossiers calculés évolue en conséquence.
- rajout de plusieurs indicateurs :
- lors du calcul de la robustesse, le nombre de taxons ayant la même valeur EK maximale est indiqué
- si la classe officielle du cours d'eau est indiqué, le programme calcule la classe de qualité correspondante, tant pour le calcul de l'IBMR que de la robustesse
- en consultation d'un dossier, la qualité de la station est mise en relief par une bordure colorée selon les règles officielles
- si les chemins d'export ne sont pas reconnus, le programme affiche maintenant un message d'information
- il est maintenant possible de rechercher des dossiers par année (les dates de recherche sont alors inopérantes)

Version 1.0.3 du 6 janvier 2017
-------------------------------
Correction de bogues :
- si une nouvelle opération de contrôle était enregistrée deux fois de suite sans fermer la fenêtre correspondante, les unités de relevé correspondantes étaient doublées, ainsi que le point de prélèvement
- certaines requêtes SQL pouvaient échouer, si elles comprenaient une clause "order by"
- l'absence de paramètres dans le fichier de configuration param.ini pouvait, dans certains cas de figure, faire planter l'application
- il n'était pas possible de supprimer une opération (contrainte d'intégrité référentielle non respectée)
- la version du runtime Java est maintenant testée, pour éviter une utilisation avec des versions obsolètes du JRE

Améliorations :
- le fichier XML d'export, et le fichier PDF généré, contiennent maintenant le numéro de la version du logiciel qui les a créé
- mise à jour de la documentation, avec en particulier la procédure à suivre pour installer une nouvelle version.
